Activation Lock Bypass for A11 - A9 Devices

Disclaimer: I cannot be held liable for any damages, charges, or other consequences resulting from the use of this guide.

This guide is intended for research purposes only. Please use it solely on devices that you legally own and have obtained through legal means.

Follow these instructions closely, any fuck-ups might make you have to redo the entire process.

Note that several services will not work. Cellular, iCloud sign-in (you can still sign into App Store to get apps), passcode, ANYTHING Secure Enclave related, booting untethered, among other things.

You will need:

- A computer, preferably on macOS. Linux should work too.

- A reliable/OEM Lightning cable.

- The locked iDevice in question

- Some sanity, just in case

Without further ado, let's get started!

1. You probably noticed that I didn't specify an iOS version. This doesn't matter, you're gonna need to restore your device anyways.

Freshly restore your device (use iTunes, Finder, or look up how to), power it on, but DO NOT connect to Wi-Fi or try to complete setup, as going to the "iPhone Locked To Owner" screen fucks with some values that we'll need.

2. Clone this git repo, cd into it using this command:

git clone https://github.com/randomuser691337/palera1n-mod-fixed/ && cd ./palera1n-mod-fixed/

LINUX USERS ONLY: You may need to run the following commands (before running the software) in a new terminal (KEEP IT OPEN!):

sudo systemctl stop usbmuxd && sudo usbmuxd -p -f

3. Now let's actually get started. We'll need to jailbreak with palera1n first to prepare the files (run these commands in the folder we CD'ed to earlier):

./palera1n.sh --tweaks [iOS version here, replace this with what's on your device e.g. 15.8]

Let the script run and follow any screen prompt if any. When it finishes you should be booted into iOS again. If for some reason you end up in recovery mode, try running the command again and it should boot you into iOS.

If everything was succesful and you booted into iOS, we are going to start the bypass process. Type the following command in the terminal:

./palera1n.sh --bypass [iOS version, same as before]

Let the process finish and your device should be in recovery mode.

Once in recovery mode, we are already done with the bypass, just run this command to boot:

./palera1n.sh --tweaks [iOS version, same as before]

DO NOT SET A PASSCODE OR SET UP TOUCH ID! This may end in constant crashes, and a setup crash loop, which would make you need to redo this whole process.

Proceed as normal, skip Wi-Fi setup, and you should be able to jailbreak with palera1n using the Tips app if wanted.

You're done! Now what?

Setting a Passcode Test (A9 devices only)

1. Complete the initial setup without setting a passcode.

2. Navigate to Settings → Touch ID and Passcode, then set a passcode.

If your device crashes or blackscreens and reboots to recovery, you won't be able to set a passcode. To recover, simply boot using the same command as before: palra1n.sh --tweaks.

If it sets normally/completes without crashing, congrats! You can set a passcode and Touch ID!

In case that the JB breaks and you need to restore it, do the following:

./palera1n.sh --restorerootfs [iOS version]

./palera1n.sh clean

./palera1n.sh --tweaks [iOS version]

./palera1n.sh --bypass [iOS version]

./palera1n.sh --tweaks [iOS version]

And you should be booted back in with a clean JB, once again!
